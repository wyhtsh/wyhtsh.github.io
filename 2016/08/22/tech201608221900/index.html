Redhat6下Oracle静默安装

2016-08-22
  • Oracle
  • Database
  • Oracle
  • Redhat

使用远程控制台对redhat进行操作安装oracle，由于没有进行图形化界面设置，学习使用命令行方法安装，将安装过程进行记录,前期设置稍稍微繁琐，其实安装起来更方便

oracle静默安装，即在没有图形化界面的情况下进行数据库的安装配置，通过安装也发现，其实就是对其中的db_install.rsp文件参数进行执行的过程，只要把该文件设置好，安装过程比图形化安装更快捷
具体过程如下

系统环境配置

主要是对操作系统的配置，最好的是建立一个单独的用户以及组，然后将oracle数据库进行单独地储存，方便日后的管理维护，这一步网上有很多相关的介绍，这里针对本人的操作做一些简单的说明。

在这阶段主要需要做以下方面的工作

  • oracle空间配置
    通过从要目录中新建一个文件夹，并单独挂载到一个盘符上，作为oracle数据库的存储空间，当然也可以再根据数据库各部分的功能进行进一步的细分，这样的效果应该会更好。暂时就在根目录下面建立/oracle作为oracle安装目录
  • oracle依赖包的安装
    oracle的安装需要相关的依赖包，若不提前安装上，在oracle的安装过程中会出现异常，因此我们需要提前将其安上，最好的方式通过yum进行安装配置，考虑到yum库的问题，需要提前检查yum库是否可用。redhat在未注册前不能使用系统提供的库,这里一个是使用安装盘中自带的本地库，另外则重新配置国内的库（163的不错），配置好了记得更新
  • 系统变量配置
    这里我们需要对oracle的安装环境进行配置，主要是系统参数配置，主机名配置，oracle路径的配置
  • 用户及组的配置
    最好是使用独立用户及组进行数据库的管理，我们这里就创建用户oracle进行数据库的管理，组的话主要有对数据库安装的组oinstall，数据库管理员dba，数据库操作oper。暂时都授权到oracle用户，最后把/oracle目录的所有权授予oracle用户。

上面就是系统相关配置，只是进行了简单的说明，详细的网上很多大家需要对具体参数进行设置的可以查询

rsp文件配置

这是最关键的一个文件参数设置，静默安装其实就是围绕该文件的设置的参数进行数据库的安装，网上也有许多大神写了相关的参数文件，后来我发现在安装包里response目录下就有db_install.rsp文件对需要配置的参数进行了介绍并有相关提示，其实我们只需要将该文件按照要求进行设置就可以了，很方便，和点下一步一样样的。

我把db_install.rsp我觉得重要的按顺序写下来供参考（有个需要注意的地方，文件中的路径都要使用绝对路，不然安装是会出错）：

  • oracle.install.responseFileVersion=/oracle/database/install/rspfmt_dbinstall_response_schema_v11_2_0
    这个我们要注意一下它的地址是否正确，注释里面说了Do not change the following system generated value. 不是说什么都不能改，install后面的不要动，前面的需要检查一下，比如我的oracle安装目录是在/oracle/database下面，因此要像上面这样写。
  • oracle.install.option=INSTALL_DB_SWONLY
    这个是设置oracle的安装方式，有三种：只安装数据库INSTALL_DB_SWONLY，安装数据库并配置INSTALL_DB_AND_CONFIG，升级数据库UPGRADE_DB。我这里只进行软件的安装，数据库的配置在安装好之后再进行详细的配置。
  • ORACLE_HOSTNAME=nx-rhel65
    系统本机名，使用命令行中使用hostname可以查看，要记得在/etc/hosts中添加
  • UNIX_GROUP_NAME=oinstall
    用户组，使用的是oracle用户所在组
  • INVENTORY_LOCATION=/oracle/u01/app/oracle/oraInventory
    指定oraInventory目录的路径，一般放在ORACLE_BASE目录下，与product平级
  • SELECTED_LANGUAGES=en,zh_CN
    语言设置，具体代码见注释
  • ORACLE_HOME=/oracle/u01/app/oracle/product/11.2.0/db_1
    ORACLE_HOME路径，之后的软件安装的位置
  • ORACLE_BASE=/oracle/u01/app/oracle
    ORACLE_BASE路径，oracle根目录
  • oracle.install.db.InstallEdition=EE
    oracle的安装版本，分别有注释，这是有EE是Enterprise Edition，即企业版
  • oracle.install.db.DBA_GROUP=dba
    dba用户组
  • oracle.install.db.OPER_GROUP=oper
    oper操作组
  • oracle.install.db.config.starterdb.type=GENERAL_PURPOSE
    创建数据库类型，用的是一般类型
  • oracle.install.db.config.starterdb.globalDBName=OrclDb11g
    数据库名
  • oracle.install.db.config.starterdb.SID=orcl
    数据库的SID
  • oracle.install.db.config.starterdb.characterSet=AL32UTF8
    数据库的字符集编码
  • oracle.install.db.config.starterdb.password.ALL=123
    数据库管理员密码设置，可以使用这个对管理员账号设置相同的密码，也可以单独设置，如果设置了相同的，后面的就可以不设置了
  • DECLINE_SECURITY_UPDATES=true
    网上说这个要设置为true，不然会报错。

主要的参数就是这些了，配好之后就可以输入命令运行了，在database下输入命令./runInstaller -silent -responseFile /oracle/database/response/db_install.rsp(responseFile后面跟的是我们需要调用的rsp文件的绝对路径，不能使用相对路径，如果想要看进度可以在其中加上-showProgress)，运行之后监控着日志，然后坐等安装结束！记得我们没有创建实例这些的，所以开始只能使用sys账号进行登录，再自行建立实例以及库。
