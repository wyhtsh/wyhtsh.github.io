Oracle 基本架构

Oracle 启动方式

Oracle在startup的时候有三种启动参数，分别是nomout、mount、open。分别是Oracle的三种启动状态。

  1. startup nomount

    在nomount状态下，Oracle启动了instance，此时没有启动数据库，或者数据库还没有被建立，这个时候可以进行控制文件的建立以及数据库的创建，此时有系统管理员可以进行操作。

  2. startup mount db

    先执行nomount
    在此状态下，Oracle可以对相应数据库db进行日志归档以及数据恢复等工作，但仍不能对数据库中的数据进行访问。

  3. startup open db

    执行nomount以及mount，打开所有数据库db的文件，可以对数据库中的数据进行访问。

  4. 单独执行startup
    单独执行startup，将按照以下顺序进行执行
    startup nomount
    alter database mount
    alter database open

不同的启动状态下，用户的执行权限不同，根据需要启动相应态度；同时也可以也可以根据启动状态判断相应的数据库故障进行初步确定。

Oracle 关闭方式

数据库的关闭方式主要有四种，分别是abort，normal，transactional，immediate
